# Contributing

Thank you for your interest in improving this project.

## Development Setup

1. Clone the repository.
2. Install Go 1.26 or newer.
3. Run tests:

```sh
go test ./...
```

## Code Style

- Keep changes focused and minimal.
- Follow existing naming and package structure.
- Format Go code before submitting:

```sh
gofmt -w .
```

## Pull Requests

1. Create a branch for your change.
2. Add or update tests when behavior changes.
3. Ensure all tests pass locally.
4. Open a pull request with:
   - A clear summary of what changed
   - Why the change is needed
   - Any compatibility considerations

## Reporting Issues

When opening an issue, please include:

- A clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Relevant logs or errors
- Go version and platform details

## Questions

If you are unsure about approach or scope, open an issue or draft PR first so we can align before large changes.
